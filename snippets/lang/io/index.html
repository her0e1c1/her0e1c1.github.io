system.scm¶

when you want to execute other commands, then use system

exec¶

execute shell command. you need -i option

$ perl -e 'system qq#sh -ic "ll /lib/libc.so.*"#'
-r--r--r--  1 root  wheel  1567216  1月 17  2014 /lib/libc.so.7

System and go last

$ perl -E 'system "echo TEST" ;say "print"'
TEST
print

コマンドを実行し、失敗した場合にのみ制御が戻る

$ perl -E 'exec "echo 1" ;say "NOT PRINTED"'
1

when you want to execute other commands, then use system

$ perl -E 'say `echo TEST`'
TEST

$ perl -e 'system "echo 1" for 1..3'
1
1
1

$ sh -c 'echo `echo TEST`'
TEST

$ sh -c 'echo $(echo TEST)'
TEST

You can nest commands if using $()

$ sh -c 'echo $(echo $(echo 1) 2) 3'
1 2 3

$ ghc -e 'System.Cmd.system $ "echo " ++ unwords [[x, '\''.'\''] ++ y | x <-['\''a'\''..'\''c'\''], y <- ["csv", "txt"]]'
a.csv a.txt b.csv b.txt c.csv c.txt
ExitSuccess

l list, v vector, e env exec path and its arguments. the list must be end with NULL.

$ ce 'execv("/bin/echo", (char*[]){"ECHO", "abc", "efg", NULL});'
abc efg

$ ce 'execv("/bin/echo", (char*[]){"/bin/echo", "abc", "efg", NULL});'
abc efg

$ ce 'execv("echo", (char*[]){"ECHO", "abc", "efg", NULL});'

you don’t need an absolute path because it searches PATH for the command.

$ ce 'execlp("echo", "ECHO", "hoge", NULL);'
hoge

$ ce 'execvp("echo", (char*[]){"ECHO", "hoge", NULL});'
hoge

$ s '(sys-system "echo 1")'

0
